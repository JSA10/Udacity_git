World Wide Web (WWW): A web of html documents

  • The WWW consists of html documents (Hyper Text Markup Language)
  • These are connected to each other using links (URLs)
  • HTML documents can contain a variety of content types: e.g. text, images, music or video.

How does the internet work?

Imagine three separate stages in a process, starting with a person on a laptop on the left and ending with giant servers that store all the html coded content previously discussed. The internet is the glue in the middle that enables web browsers on people's laptops to connect with and find content stored on these servers. Content is created and structured using html and url link identifiers enable location. A technology called "Hyper Text Transfer Protocol" (HTTP) enables the internet to communicate with the servers.

HTTP forms the connection with servers, allowing individual web browsers to find and translate the files stored on them into content that can be displayed for and used by people.

Understanding HTML

HTML is made up of four elements:

  1. Text content (that you see on a web page)
  2. Markups (code that determines what it looks like)
  3. References to other documents (images / videos)
  4. Links to other pages

HTML tags:

Structure: Tags consist of an opening tag, followed by visible content, closed with a matching second tag. Each pair of opening and closing tags can be thought of as an element.

Attributes: Tags can have attributes, a common example is the anchor tag, which can be assigned links.

Attributes: Tags can have attributes, a common example is the anchor tag, which can be assigned links.

Void Tags: Image tags are examples of void tags. Void tags have no text 'content' so don't need closing tag. NOTE: Image tags contain two attributes: A URL link to actual image, plus an alternative message in case image doesn't load, or as a marker for the blind.

Whitespace: Multiple lines in html code will make no difference to output as all whitespace is ignored (but for a single standard space in the text). Use break to insert a break (another void tag). Can also use the paragraph tag.

Inline vs. Block: HTML code can work inline, within the text itself e.g. break, emphasis or image. Or it can effectively place a wrapper around a block of content e.g. paragraph.

Container tags: These are tags that don't do anything themselves but contain content and attributes. Examples are span tag (inline) or div tag (block). Both can have attributes added to them, like css code to change style and design.

For more on HTML tags, see w3schools list split by category / usage

Examples of inline vs. block tags:

  • Inline: anchor, span, break, image, and strong
  • Block: while divide, paragraph, and form

CSS: Cascading Style Sheets

Introduction to using CSS

What is CSS? CSS is code written to control the "style" of HTML elements.

How does CSS "control" HTML? If you want to write CSS that makes all h1 elements have a black background and white text, you would write:

h1 { background-color : black; color: white; }

In this example, the h1 is a selector. It says to the browser "I want you to apply the rules I'm about to tell you to every h1 element." The line of code that says background-color : black; is a declaration. background-color is a property and black is the value of that property.

To write a comment for other readers of your code, you'd write: /* This is a CSS comment.*/

CSS Vocabulary

CSS: Cascading Style Sheets. Named as they are made up of sheets that shape the style of a page and can be structured in a cascading manner. Imagine a tree structure, but key difference is specifications within one can be applied to all, or focused to specific sections. Style: This word can refer to many things and so it can be confusing. It can refer to: The HTML element. For example: The HTML attribute. For example:

The general look of a web page. For example: "I like that site's style." A verb. For example: "I'm making progress on my page. The structure is all done but now I have to style it." Rule: a line of CSS code describing the value that a certain attribute should take. Property: The property you want to change. Value: The value that you want to assign to the attribute. Selector: The name that you use in order to target the elements that are assigned to a class or id attribute in the HTML. Class: A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="1.1" ID: ID's are unique identifiers that uniquely identifies an element in HTML.
