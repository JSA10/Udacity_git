Lesson 1: The basics of the web

World Wide Web (WWW): A web of HTML documents

  • The WWW consists of HTML documents (Hyper Text Markup Language)
  • These are connected to each other using links (URLs)
  • HTML documents can contain a variety of content types: e.g. text, images, music or video.

How does the internet work?

In the process depicted below, the internet is the glue in the middle that enables web browsers on people's laptops to connect with and find content stored on these servers. Content is created and structured using HTML, each piece has a URL link that acts as an identifier. A technology called "Hyper Text Transfer Protocol" (HTTP) enables the internet to communicate with the servers.

Understanding HTML

HTML is made up of four parts:

  1. Text (that you see on a web page)
  2. Markups (code that determines what the page looks like)
  3. References to other documents (images / videos)
  4. Links to other pages

HTML tags:

Structure: Tags consist of an opening tag, followed by visible content, closed with a matching second tag. Each pair of opening and closing tags can be thought of as an element.

Attributes: Tags can have attributes, a common example is the anchor tag, which can be assigned links.

Void Tags: Image tags are examples of void tags. Void tags have no text 'content' so don't need closing tag.

Whitespace: Multiple lines in HTML code will make no difference to output as all whitespace is ignored (but for a single standard space in the text). Paragraph (p) tag is the best tag to use for line breaks as enables consistent sections and format of text. Can use break (br) too, which is another void tag.

Inline vs. Block: HTML code can work 'inline'; within the text itself e.g. break, emphasis or image. Or it can effectively place a wrapper around a 'block' of content e.g. paragraph.

Container tags: These are tags that don't do anything themselves but contain content and attributes. Examples are span tag (inline) or div tag (block). Both can have attributes added to them, like css code to change style and design, or reference IDs.

For more on HTML tags, see w3schools list split by category / usage

Examples of inline vs. block tags:
  • Inline: anchor, span, break, image, and strong
  • Block: while divide, paragraph, and form

Lesson 2: Creating structure in a website with HTML

HTML, CSS and Javascript are building blocks of websites

Think of a website as a house:

HTML is the structure - where the walls are, how many rooms etc. CSS is the style - the colours of the walls, what wallpaper or flooring there is Javascript adds interaction - things that can change things within your house - e.g. tv remote or thermostat

Inspecting live HTML with dev tools

When exploring a web page using the Safari developer tool 'Web Inspector' we can observe that:

  1. All HTML is rectangular
  2. We can drill down to the actual text that is displayed on the page
  3. By using the style panes we can see formating parameters - fonts and borders etc.

HTML follows a tree like structure

HTML code works best when grouped into sections, with each each new section traceable back to a 'parent' branch above

HTML-CSS-DOM

  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks)
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page

HTML is the language of the web, with set syntax and rules. HTML uses tags to form elements, these are structured like a tree and tell the browser what to display.

CSS also follows syntax and rules. CSS is used within the struture of HTML tags to define style.

Everything in HTML is boxes

Many options to combine to make very 'unboxlike' things but need to understand the principle and the need to 'boxify' design.

Start with a plan, then build structure:
  1. Go old school - start by drawing key points and structure on paper, much like an essay or project plan
  2. Split the paper page into boxes and annotate what goes into them
  3. When get to setting out in HTML use container tags e.g. <div class = "label" > content </div>.

    NOTE: The class attribute is metadata describing what this container or box holds

Lesson 3: CSS = Cascading Style Sheets

Introduction to using CSS

NOTE: The original source of CSS intro and vocabulary is Andy's notes, I've added and tweaked in a few places.

What is CSS?

CSS is code written to control the "style" of HTML elements.

How does CSS "control" HTML?

If you want to write CSS that makes all h1 elements have a black background and white text, you would write:

h1 {
background-color : black;
color: white;
}

In this example, the h1 is a selector. It says to the browser "I want you to apply the rules I'm about to tell you to every h1 element." The line of code that says background-color : black; is a declaration. background-color is a property and black is the value of that property.

To write a comment for other readers of your code, you'd write: /* This is a CSS comment.*/

CSS Vocabulary

CSS: Cascading Style Sheets. Named as they are made up of sheets that shape the style of a page and can be structured in a cascading manner. Imagine a tree structure, but key difference is specifications within one can be applied to all, or focused to specific sections.

Style: This word can refer to many things and so it can be confusing. It can refer to:
  • The HTML element. For example: <style>div {color:blue}</style>
  • The HTML attribute. For example: <div style="color : blue">this text would be blue</div>
  • The general look of a web page. For example: "I like that site's style."
  • A verb. For example: "I'm making progress on my page. The structure is all done but now I have to style it."

Rule: a line of CSS code describing the value that a certain attribute should take.

Property: The property you want to change.

Value: The value that you want to assign to the attribute.

Selector: The name that you use in order to target the elements that are assigned to a class or id attribute in the HTML.

Class: A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="l1.1"

ID: ID's are unique identifiers that uniquely identifies an element in HTML.

Everything in HTML is boxes

Therefore box size calculations are important. However, box size doesn't just include the content itself, but also three outer layers: padding, border and margin. See visual example here:

You can specify size of individual elements, but this can become complex so often best to set the box-sizing element to border box for every element:

*{
box-sizing: border-box;
}

NOTE: This is a new feature so need to ensure it works across browsers by specifying them in advance. Do this by referencing webkit (safari and chrome), moz or ms.

Complete CSS code would be:

*{
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

Scaling:Using pixels to size boxes means that the parameters will never change, which could lead to a bad user experience as screen sizes change. Can also use a % figure which will evolve.

Code-Test-Refine

Throughout this lesson we have been following a simple process of Code-Test-Refine.

A process for creating a web page:

  • Start by defining the page structure
  • Map the design - visually boxify (go old school with paper)
  • Add HTML tags to replicate structure
  • Size the boxes throughout
  • Position the boxes
  • Markup the final details

5 steps to translating a pre-existing design

When translating a design use a 5 step process:

  1. Look for natural boxes
  2. Look for repeated styles and semantic elements (e.g. H2s)
  3. Write your HTML
  4. Apply styles
  5. Fix things
During the process, repeat steps 3 and 4 until code replicates the design exactly
